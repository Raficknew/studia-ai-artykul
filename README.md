# Grupowy artykul na temat AI

## Cel zadania
Celem jest przeprowadzenie pelnego cyklu pracy grupowej w GitHub: od forkowania repozytorium, przez prace na galeziach, po recenzje i rozwiazywanie konfliktow.

## Tematyka artykulu (do wyboru przez sekcje w grupie)
Artykul powinien byc napisany w Markdown i opierac sie na zrodlach. Mozliwe obszary:

- Agentic AI: automatyzacja i orkiestracja w cyklu zycia oprogramowania
- Wplyw narzedzi AI na produktywnosc: ekosystem GitHub Copilot i GitHub Spark
- Wyzwania etyczne i prawne: Generative AI a licencje Open Source
- Transformacja bezpieczenstwa kodu dzieki sztucznej inteligencji
- Przyszlosc systemow kontroli wersji w erze inteligentnej orkiestracji

## Instrukcja techniczna (workflow)
1. Setup (lider zespolu)
   - Stworzenie publicznego repozytorium na GitHub i dodanie pliku README.md oraz szkieletu artykulu AI_article.md
   - Zaproszenie czlonkow zespolu lub udostepnienie linku do forkowania
2. Praca rozproszona (czlonkowie zespolu)
   - Fork i clone repozytorium lidera
   - Praca na osobnych galeziach dedykowanych sekcjom artykulu (np. feature/legal-issues)
   - Regularne commity z jasnymi opisami
3. Integracja i code review
   - Push do forka i otwarcie Pull Request do repozytorium lidera
   - Kazdy PR musi byc zrecenzowany przez przynajmniej jednego studenta
   - Wprowadzenie poprawek az do uzyskania akceptacji
4. Finalizacja
   - Merge zaakceptowanych PR do galezi main
   - W razie konfliktow wspolne rozwiazanie konfliktow
